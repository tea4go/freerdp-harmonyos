## 1. 开发环境配置

### 1.1 必要软件

| 软件 | 版本要求 | 用途 |
|------|----------|------|
| DevEco Studio | 5.0.0+ | HarmonyOS IDE |
| HarmonyOS SDK | API 12+ | 开发套件 |
| Node.js | 18+ | 构建工具依赖 |
| Git | 最新版 | 版本控制 |

### 1.2 DevEco Studio 配置

1. **下载安装 DevEco Studio**
   - 官网: https://developer.harmonyos.com/cn/develop/deveco-studio
   - 选择 Windows 版本下载

2. **安装 HarmonyOS SDK**
   ```
   DevEco Studio → Settings → SDK → HarmonyOS SDK
   勾选 API 12 (HarmonyOS 5.0.0)
   ```

3. **配置 NDK (Native 开发)**
   ```
   SDK 管理器 → HarmonyOS SDK → NDK
   安装 Ohos NDK (用于编译 C/C++ 代码)
   ```

4. **配置签名证书**
   ```
   File → Project Structure → Signing Configs
   选择自动签名或手动配置证书
   ```

### 1.3 项目配置检查

确保以下文件配置正确：

**build-profile.json5**:
```json
{
  "apiType": "stageMode",
  "buildOption": {
    "externalNativeOptions": {
      "path": "./src/main/cpp/CMakeLists.txt",
      "cppFlags": "-O2 -DOHOS_PLATFORM",
      "abiFilters": ["arm64-v8a"]
    }
  }
}
```

## 2. 项目运行

### 2.1 克隆项目

```bash
git clone https://github.com/your-repo/freerdp-harmonyos.git
cd freerdp-harmonyos
```

### 2.2 打开项目

1. 启动 DevEco Studio
2. 选择 `Open` 打开项目目录
3. 等待项目同步完成 (首次可能需要下载依赖)

### 2.3 连接设备

**方式一: 真机调试**
1. 开启开发者模式
2. USB 连接设备
3. 在设备上授权调试

**方式二: 模拟器**
```
DevEco Studio → Tools → Device Manager
创建或启动模拟器 (注意: 模拟器不支持 Native 库调试)
```

> ⚠️ **注意**: 由于依赖 Native 库，建议使用真机调试。模拟器可能无法加载 .so 文件。

### 2.4 运行应用

1. 选择运行配置 (entry)
2. 点击运行按钮 (Shift+F10)
3. 等待编译和安装完成

### 2.5 快速调试连接

项目内置了自动连接功能，方便调试：

```typescript
// entry/src/main/ets/pages/Index.ets
@State autoConnectOnStart: boolean = true;  // 启动时自动连接

// 默认连接配置
quickConnectHost: string = '192.168.52.127';
quickConnectPort: string = '3389';
quickConnectUser: string = 'administrator';
quickConnectPassword: string = 'testing123';
```

修改这些值可以快速测试不同的连接配置。

### 3.1 日志调试

**ArkTS 日志**:
```typescript
console.info('TAG: 信息日志');
console.warn('TAG: 警告日志');
console.error('TAG: 错误日志');
console.debug('TAG: 调试日志');
```

**Native 日志 (C++)**:

```bash

```

**ArkTS 断点**:
1. 在代码行号左侧点击设置断点
2. 使用 Debug 模式运行 (Shift+F9)
3. 断点命中时查看变量值

**Native 断点**:
1. 在 C/C++ 代码中设置断点
2. 创建 Native 调试配置
3. 附加到进程进行调试

### 3.3 网络调试

**检查网络连接**:
```typescript
// 使用 NetworkManager 检查网络状态
const isAvailable = NetworkManager.isNetworkAvailable();
console.info(`网络可用: ${isAvailable}`);
```

**抓包分析**:
```bash
# 使用 Wireshark 抓取 RDP 流量
# 过滤器: tcp.port == 3389
```

### 3.4 Native 库加载调试

**检查库是否加载**:
```typescript
if (!LibFreeRDP.isNativeLoaded()) {
  const error = LibFreeRDP.getLoadError();
  console.error(`Native 库加载失败: ${error}`);

```

**常见加载失败原因**:
1. .so 文件架构不匹配 (需要 arm64-v8a)
2. 依赖库缺失
3. OpenSSL 配置问题

### 3.5 连接流程调试

启用详细日志查看连接过程：

```typescript

  asyncUpdate: true,
  debugLevel: 'DEBUG'  // 或 'TRACE'

```

查看连接事件顺序:

## 4. 常见问题

### 4.1 Native 库加载失败

**现象**: 显示 "原生库加载失败" 对话框

**原因**:
- 使用了 glibc 编译的库 (HarmonyOS 使用 musl libc)
- 架构不匹配 (需要 arm64-v8a)

**解决方案**:
```bash
# 确认 .so 文件架构
file libs/arm64-v8a/libfreerdp3.so
# 应显示: ELF 64-bit LSB shared object, ARM aarch64

# 使用 HarmonyOS NDK 重新编译 FreeRDP
# 参考: scripts/build_freerdp.sh
```

### 4.2 连接成功后崩溃

**现象**: 连接建立后应用立即退出

**常见原因**:
1. 回调函数未正确设置
2. 空指针访问
3. 线程安全问题

**调试方法**:
```cpp
// 在 harmonyos_freerdp.cpp 中添加日志
OH_LOG_INFO(LOG_APP, "连接回调开始");
// 检查每个回调是否正常
```

### 4.3 OpenSSL 初始化失败

**现象**: SIGABRT 崩溃，日志显示 OpenSSL 错误

**原因**: OpenSSL 尝试从不存在的路径加载配置

**解决方案** (已在 harmonyos_napi.cpp 中实现):
```cpp
// 必须在任何 OpenSSL 调用之前设置
setenv("HOME", "/data/storage/el2/base/files", 1);
setenv("OPENSSL_CONF", "/dev/null", 1);
unsetenv("OPENSSL_MODULES");
unsetenv("OPENSSL_ENGINES");
```

### 4.4 后台音频中断

**现象**: 锁屏后音频停止

**检查清单**:
1. 确认 `ohos.permission.KEEP_BACKGROUND_RUNNING` 权限
2. 确认后台服务已启动
3. 确认使用了 `AUDIO_PLAYBACK` 后台模式

**调试**:
```typescript
console.info(`后台服务状态: ${RdpBackgroundService.isServiceRunning()}`);
```

### 4.5 网络断开后无法重连

**现象**: WiFi 切换或信号弱时连接断开，无法自动恢复

**检查**:
1. NetworkManager 是否正确初始化
2. 重连回调是否设置
3. 最大重试次数 (10次)

**手动触发重连**:
```typescript
NetworkManager.startReconnect();
```

### 4.6 画面卡顿或静止

**现象**: 远程桌面画面不更新

**可能原因**:
1. 后台模式未正确退出
2. 图形解码被禁用
3. 网络延迟过高

**解决**:
```typescript
// 从后台返回时
onPageShow(): void {
  if (this.session && this.session.getInstance() !== 0) {
    // 恢复图形解码
    LibFreeRDP.setClientDecoding(this.session.getInstance(), true);
    // 请求全屏刷新
    LibFreeRDP.requestRefresh(this.session.getInstance());
  }
}
```

### 4.7 触摸输入不响应

**现象**: 点击远程桌面没有反应

**检查**:
1. SessionView 的 onTouch 事件是否绑定
2. 坐标转换是否正确
3. FreeRDP 实例是否有效

**调试**:
```typescript
// 在 SessionView 中添加日志
private onTouchDown(event: TouchEvent): void {
  const pos = this.viewToDesktop(touch.x, touch.y);
  console.debug(`触摸位置: ${pos.x}, ${pos.y}`);
  this.sendMouseEvent(pos.x, pos.y, PTR_FLAGS_DOWN | PTR_FLAGS_BUTTON1);
}
```

## 5. 性能调优

### 5.1 图形性能

**启用硬件加速**:
```typescript
// 使用 H.264 硬件解码 (如果设备支持)
performanceFlags: {
  h264: LibFreeRDP.hasH264Support(),
  // ...
}
```

**调整刷新策略**:
```typescript
// 后台时禁用图形
LibFreeRDP.setClientDecoding(instance, false);

// 前台时启用并刷新
LibFreeRDP.setClientDecoding(instance, true);
LibFreeRDP.requestRefresh(instance);
```

### 5.2 网络性能

**TCP Keepalive 配置**:
```typescript
// 15秒间隔，适合 NAT 穿透
LibFreeRDP.setTcpKeepalive(instance, true, 15, 15, 3);
```

**心跳间隔**:
```typescript
// 30秒 RDP 心跳
const HEARTBEAT_INTERVAL_MS = 30000;
```

### 5.3 内存优化

**及时释放资源**:
```typescript
// 释放 PixelMap
if (this.pixelMap) {
  this.pixelMap.release();
  this.pixelMap = null;
}

// 清理会话
GlobalSessionManager.cleanupAll();
```

**避免内存泄漏**:
```typescript
// 组件销毁时清理
aboutToDisappear(): void {
  this.cleanup();
  // 清除所有定时器
  clearInterval(this.heartbeatTimer);
  clearTimeout(this.toolbarHideTimer);
}
```

### 5.4 性能监控

**监控连接健康**:
```typescript
const health = LibFreeRDP.getConnectionHealth(instance);
// -1: 无效, 0: 断开, 1: 降级, 2: 健康

const idleTime = LibFreeRDP.getIdleTime(instance);
// 检查空闲时间，判断连接是否活跃
```

**监控帧率**:
```typescript
// 在 SessionState 中记录更新
recordUpdate(): void {
  this.lastUpdateTime = Date.now();
  this.updateCount++;
}

// 计算帧率
const fps = this.updateCount / (this.getConnectionDuration() / 1000);
```

## 6. 日志系统

### 6.1 日志级别

| 级别 | 用途 |
|------|------|
| TRACE | 详细跟踪信息 |
| DEBUG | 调试信息 |
| INFO | 一般信息 |
| WARN | 警告信息 |
| ERROR | 错误信息 |

### 6.2 配置日志级别

**ArkTS 层**:
```typescript
debugSettings: {
  debugLevel: 'DEBUG'  // TRACE | DEBUG | INFO | WARN | ERROR
}
```

**Native 层**:
```cpp
// 在编译时设置
#define DEBUG_LEVEL LOG_DEBUG
```

### 6.3 日志输出位置

**DevEco Studio**:
- Logcat 窗口 → 选择设备和进程
- 过滤 TAG: `FreeRDP` 或 `FreeRDP.NAPI`

**命令行**:
```bash
# 实时日志
hdc shell hilog -T FreeRDP

# 保存到文件
hdc shell hilog > debug.log
```
